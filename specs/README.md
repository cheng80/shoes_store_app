# 📚 프로젝트 문서 관리

## 📁 문서 위치 규칙

**⚠️ 중요: 모든 개발 관련 md 문서는 `specs` 폴더 안에서만 생성합니다.**

- 절대 다른 폴더에는 md 문서를 생성하지 않습니다.
- 모든 설계, 진행 상황, 참고 자료는 이 폴더 내에서 관리합니다.

## 📄 문서별 용도

### 📁 프로젝트 관리 문서

#### `README.md` (현재 문서)

- 프로젝트 전반적인 요약을 기록합니다.
- 문서 관리 규칙 및 프로젝트 개요를 포함합니다.

#### `PROGRESS.md` (완료 기록)

- **목적**: 완료된 작업의 상세 기록 및 이력 관리
- **내용**:
  - 완료된 기능의 상세 설명
  - 기술적 개선 사항 기록
  - 최근 완료된 작업 (날짜별)
  - 구현 세부사항 및 변경 이력
- **사용 시점**: 작업 완료 후 상세 내용 기록
- **특징**: 과거 지향적, 이력 중심

#### `TODO.md` (작업 계획)

- **목적**: 작업 계획 및 진행 상황 추적
- **내용**:
  - 할 일 목록 (체크박스 형식)
  - 우선순위별 작업 분류
  - 예정된 기능 및 향후 고려 사항
  - 알려진 이슈
- **사용 시점**: 작업 계획 수립, 진행 상황 업데이트
- **특징**: 미래 지향적, 계획 중심

**관계**:
- `TODO.md`에서 작업 완료 시 체크박스 체크 → 간단한 완료 표시
- `PROGRESS.md`에 상세 내용 기록 → 완료된 작업의 이력 관리
- 두 문서는 서로 보완적 관계 (중복이 아닌 역할 분리)

### 📁 데이터베이스 문서

#### `DATABASE_GUIDE.md` ⭐ **시작하기**

- 데이터베이스 구조, 핸들러 사용법, 쿼리 작성 가이드를 통합한 실용 가이드입니다.
- DB 초기화, CRUD 작업, 조인 쿼리, 더미 데이터 사용법을 포함합니다.
- **개발 시 가장 먼저 참고해야 할 문서입니다.**

#### `DATABASE_SCHEMA.md`

- 데이터베이스 스키마 상세 정보를 기록합니다.
- 테이블 구조, 관계, 인덱스 등 상세한 스키마 정보를 포함합니다.

#### `HANDLER_USAGE_GUIDE.md`

- 데이터베이스 핸들러 사용법 상세 가이드입니다.
- 각 핸들러의 메서드, 사용 예제, 베스트 프랙티스를 포함합니다.

#### `DB_QUERY_COMPARISON.md`

- `db.query`와 `db.rawQuery` 비교 가이드입니다.
- 각각의 사용 시나리오와 예제를 포함합니다.

### 필수 워크플로우

**모든 작업은 문서 기반 개발 워크플로우를 따라야 합니다:**

1. **작업 시작 전**: 관련 설계 문서, TODO, REFERENCE 확인
2. **작업 진행 중**: 문서 요구사항과 일치하는지 확인하며 작업
3. **작업 완료 후**: 문서 갱신 필요 여부 확인 및 업데이트
